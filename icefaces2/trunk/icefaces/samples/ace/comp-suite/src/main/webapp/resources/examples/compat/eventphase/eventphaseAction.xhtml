Demo Use Cases:

1. Leave the inputText field blank and press the "Normal Action" button.

You will notice the validation fires as the normal JSF lifecycle is being completed.

2. Leave the inputText field blank and press the "Set Event Phase Action" button.

In the Event Log you will see that the actionListener method was reached in the bean and no validation fired.

This is because setEventPhase has moved that actionListener from the end of the lifecycle to the start, before validation fires.

This is similar to adding the immediate="true" attribute to a component, but allows greater customization.

3. Uncheck "Enable Set Event Phase" and press "Apply".

4. Leave the inputText field blank and press the "Set Event Phase Action" button.

You can see that without setEventPhase the button is the same as "Normal Action" and will fire validation which blocks the actionListener from executing.
