Contents

Both Binary and Source Bundles are available for download. The contents of each bundle type vary, as described below.


Binary Bundle


The binary bundle comes with the ICEfaces libraries pre-compiled and the sample applications pre-built to run on "Servlet-profile" platforms, such as Tomcat. The binary bundle also includes all the source files for the sample applications, but not the ICEfaces libraries themselves. 

The icefaces directory of the binary bundle contains:
  • docs
    • The ICEfaces API and TLD documentation.
      • ace - JavaDocs and TLD docs for the ICEfaces Advanced Components (api and tld).
      • compat - JavaDocs and TLD docs for the ICEfaces 1.8 compatibility APIs (compat/api), and the ICEfaces Components (compat/comps/api, compat/comps/tld).
      • core - JavaDocs for the core framework APIs.
  • lib
    • The ICEfaces libraries along with any possibly required 3rd-party run-time libraries.
      • icefaces.jar - The core ICEfaces framework.
      • icepush.jar - The ICEpush library, provides push functionality to the ICEfaces framework (optional).
      • icefaces-compat.jar - ICEfaces compatibility library that includes the original suite of ICEfaces components and a set of ICEfaces 1.x core APIs to ease migration of existing ICEfaces 1.x applications to ICEfaces 2.0. (optional).
      • icefaces-ace.jar - The ICEfaces Advanced Components library (optional).
  • licenses
    • Summary of legal licenses used by ICEfaces and 3rd party libraries.
  • maven2
    • Project files to support using ICEfaces with Maven.
  • samples
    • ace
      • Examples related to the ICEfaces Advanced Component Environment (ACE).
      • See the Advanced Components topic of the ICEfaces Wiki for more information.
        • /showcase
          • ICEfaces Advanced Components library showcase sample application.
    • compat
      • ICEfaces 1.x sample applications that have been ported to ICEfaces 2 using the ICEfaces Components and the core compatibility APIs.
      • See the Compatibility and ICEfaces Components topics in the ICEfaces Wiki for more information.
        • /auctionMonitor
        • /component-showcase
    • core
      • ICEfaces 2.0 core framework sample applications. Demonstrates using the ICEfaces framework without the ICEfaces Components or Advanced Components.
        • /auction
          • A simulated online auction application using only standard JSF components. Demonstrates core benefits of the ICEfaces framework, including the use of push to asynchronously update user's browsers when an auction bid occurs.
        • /portlets/chat
          • A simple ICEfaces chat portlet that demonstrates using ICEfaces and ICEpush in a portal environment.
    • dist
      • Contains all the sample app .wars prebuilt for "Servlet-profile" platforms, such as Tomcat.

Source Bundle


The source bundle contains all the same libraries and applications as the binary bundle but also includes the complete source code for the ICEfaces libraries themselves, and additional required ICEfaces build-time libraries.  The source bundle does not contain any pre-built ICEfaces libraries or example .wars.

Use the "ant -p" command in the "icefaces" directory to see the available ant build targets.

The icefaces directory of the source bundle contains:
  • ace
    • Source files and resources for the ICEfaces Advanced Component Environment (ACE) and components
    • See the Advanced Components topic of the ICEfaces Wiki for more information.
  • compat
    • Source files and resources for the ICEfaces Components and the ICEfaces 1.x compatibility APIs.
    • See the Compatibility and ICEfaces Components topics in the ICEfaces Wiki for more information.
  • core
    • Source files and resources for the ICEfaces core framework.
  • lib
    • The ICEfaces libraries are located here (after being built) along with any possibly required 3rd-party run-time and build-time libraries.
      • icefaces.jar - The core ICEfaces framework.
      • icepush.jar - The ICEpush library, provides push functionality to the ICEfaces framework (optional).
      • icefaces-compat.jar - ICEfaces compatibility library that includes the original suite of ICEfaces components and a set of ICEfaces 1.x core APIs to ease migration of existing ICEfaces 1.x applications to ICEfaces 2.0. (optional).
      • icefaces-ace.jar - The ICEfaces Advanced Components library (optional).
  • licenses
    • Summary of legal licenses used by ICEfaces and 3rd party libraries.
  • maven2
    • Project files to support using ICEfaces with Maven.
  • samples
    • ace
      • Examples related to the ICEfaces Advanced Component Environment (ACE).
      • See the Advanced Components topic of the ICEfaces Wiki for more information.
        • /showcase
          • ICEfaces Advanced Components library showcase sample application.
    • compat
      • ICEfaces 1.x sample applications that have been ported to ICEfaces 2 using the ICEfaces Components and the core compatibility APIs.
      • See the Compatibility and ICEfaces Components topics in the ICEfaces Wiki for more information.
        • /auctionMonitor
        • /component-showcase
    • core
      • ICEfaces 2.0 core framework sample applications. Demonstrates using the ICEfaces framework without the ICEfaces Components or Advanced Components.
        • /auction
          • A simulated online auction application using only standard JSF components. Demonstrates core benefits of the ICEfaces framework, including the use of push to asynchronously update user's browsers when an auction bid occurs.
        • /portlets/chat
          • A simple ICEfaces chat portlet that demonstrates using ICEfaces and ICEpush in a portal environment.
