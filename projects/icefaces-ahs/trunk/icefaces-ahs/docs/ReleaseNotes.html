Understanding ICEfaces Asynchronous HTTP Server (AHS)

ICEfaces Asynchronous HTTP Server is open-source software licensed under the Mozilla Public License (MPL) 1.1.

ICEfaces Asynchronous HTTP Server provides enhanced scalability and robustness over ICEfaces. ICEfaces Asynchronous HTTP Server provides the following additional features:

  • Asynchronous HTTP Server: Provides Enterprise-level scalability and robustness for ICEfaces applications that leverage server-initiated rendering and asynchronous mode deployments.
  • Clustered Deployment Support: ICEfaces applications can now be deployed in clustered environments. ICEfaces Asynchronous HTTP Server includes documented configurations for clustered deployments of ICEfaces applications.
    • Note: Clustered deployment testing for ICEfaces AHS is limited to the JBoss 4.0, BEA WebLogic 8.1 SP 4, and WebSphere 6.0.2 application servers. Future releases will include tested configurations for all supported application servers.

New and Notable

The following notable new features are introduced in this release:

ICEfaces Asynchronous HTTP Server

  • Improved threading model for JMS communication.
  • Fixed a socket leak issue where sockets could end up in a CLOSE_WAIT state, without actually being closed.
  • Minor bug fixes.

Prerequisites

ICEfaces 1.6.1 is required in order to use the ICEfaces Asynchronous HTTP Server 1.6.1a.

Supported Platforms

J2EE Application Servers

Enterprise deployments of ICEfaces applications require the Java Messaging Service (JMS), which means that Enterprise deployments require a J2EE compliant application server. The ICEfaces framework supports the following applications servers for development of ICEfaces applications. This support is applicable to both ICEfaces CE and ICEfaces AHS.

Vendor Product Version
Apache Tomcat with ActiveMQ 5.0.28, 5.5
JBoss Inc. JBoss Application Server 4.0.3
BEA Systems Inc. Weblogic Server 8.1
Sun Microsystems Sun Java System Application Server 8.1
Oracle Oracle Application Server Container for J2EE (OC4J) 10.1.3
IBM Websphere Application Server 6.0.2

For Enterprise deployments that utilize the Asynchronous HTTP Server, and/or clustering, the following restrictions apply:

  • Tomcat is not a supported environment as JMS is required but not supported in Tomcat.
  • Current testing of Asynchronous HTTP Server deployment is limited to JBoss, WebLogic, and WebSphere for this release.
  • Current testing of clustered deployments is limited to JBoss, WebLogic, and WebSphere for this release.

Known Issues

The following are known issues with this release:

Application Server

  • When shutting down or undeploying an ICEfaces application, the application server may log exceptions related to the classloader unloading classes before framework threads have finished running.
